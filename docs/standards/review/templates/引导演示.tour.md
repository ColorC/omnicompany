# 引导演示 TourScript 模板

> 用法: 复制下面 JSON 骨架, 填成 app 内 `src/.../demo-tours.ts` 的一条 tour。
> 规范见 [../引导演示材料规范.md](../引导演示材料规范.md)。schema 权威 = wiki-core `demo-script.d.ts`。

## 骨架(复制填写)

```jsonc
{
  "id": "<tourId, 短横线命名, 如 r2x-newbie>",
  "title": "<导览标题(中文)>",
  "app": "<app 名, 如 walker-game>",
  "version": 1,
  "steps": [
    {
      "id": "<稳定步骤 id, 如 r2x-01-setup>",
      "title": "<步骤标题(中文)>",
      "narration": "<这一步在讲什么、用户该看哪(中文)>",
      "target": "<CSS 选择器, 优先用已有 data-testid>",
      "action": { "type": "waitFor", "target": "<选择器>" },
      "screenshotRef": "beat-01-<name>",
      "links": [
        { "rel": "doc_paragraph", "href": "wiki://<page.md>#h=<fnv1a-hash>", "label": "<说明>" }
      ]
    }
  ]
}
```

## action 取值

| type | 字段 | 含义 |
|---|---|---|
| `click` | `target` | 点一个 CSS 选择器元素 |
| `clickCell` | `q`,`r` | 点棋盘投影格(app 注册的 hook) |
| `waitFor` | `target`,`timeoutMs?` | 等元素出现 |
| `waitMs` | `ms` | 固定等待 |
| `eval` | `ref` | 调 app 注册的 actionHook(禁裸串) |

## 落地清单(缺一不可)

1. tour 写进 app `src/.../demo-tours.ts`, `?demo=<tourId>` 可挂载。
2. 录制: `node tools/ops/ops-run-demo-tour.mjs <tourId>` → `docs/reviews/<date>/demo-<tourId>.webm` + 帧 + manifest。
3. 提交为 `html` 材料 `extra.live_url=/<app>/?...&demo=<tourId>`, 并在父 webgame-spec 材料 `extra.demo` 引用之。
4. narration/title 中文-only。
