# Reviewable 导览模板

> 用法: 每个推给用户审阅的 reviewable(material)按此填三段。提交 / push 时作为元数据带上。
> 规范见 [../审阅与推送规范.md](../审阅与推送规范.md)。三段缺一不可——缺了就别推。

---

## 模板(复制填写)

```
### 📍 对应需求
<这产物满足哪条需求 / 用户故事。给出处, 例: 用户故事_增强版.md US-4.2 / 原始需求 §4.6>

### ✅ 完成度
- 状态: <完成 | 部分完成 | 阻塞>
- 做了: <具体做到了什么>
- 没做完 / 已知缺陷: <还差什么、哪里是半成品、有什么已知 bug>

### 🧭 体验路径
1. <从哪进>
2. <点/看什么>
3. <预期看到什么 = 验收点>
<若"暂无可审产物"或"是测试夹具", 在此显式说明, 不要冒充真产物>
```

---

## 填写示例

```
### 📍 对应需求
用户故事_增强版.md US-4.6(网页产物是"真内容", 过程说明只进批注) + 原始需求 §4.2.3(网页可审阅)。

### ✅ 完成度
- 状态: 部分完成
- 做了: 网页本体渲染 + 按元素圈选评论 + 裁决(accept/reject/block)。
- 没做完: 网页内"AI 说明"侧栏未接;批注与正文分离的校验未做。

### 🧭 体验路径
1. 审阅台 → 必验收区 → 打开本 material。
2. 在第二段文字上圈选 → 写一条评论。
3. 预期: 评论锚定到该元素、出现在批注列表、可发送给总控。
   验收点 = 评论确实锚到元素 + 能送达。
```
