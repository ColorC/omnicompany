# 引导演示材料规范 · 2026-06-13

> 来源: 用户 2026-06-13 对审阅材料格式不满, 裁决"主体型材料(webgame-spec)的主审材料应是标准化的引导动画/演示流程,
> 逐步引导用户过完所有修改, 每步可评论, 评论伴审批存成修改意见"。本规范是引导演示这一审阅形态的唯一权威。
> 上位: [review_report_form_association.md](../protocol/review_report_form_association.md)(主体类型↔形态关联)。

## 一句话

**webgame(及一切 web/图形化软件)类审阅, 主审材料 = 一套基于标准化演示设施的引导演示**——驱动 app 的**真实 UI**, 用覆盖层逐步引导用户过完本次所有修改, **每一步可评论**, 评论伴审批保存即成"修改意见"。不是录屏甩给用户, 不是在 dashboard 里把烘焙好的帧重渲一遍。

## 为什么

主体型产物(游戏/软件)的"主体"是它本身的运行态。脱离真实 UI 的截图堆叠或信息面板平铺, 传递率低且容易"测试绿≠体验成立"。引导演示让 agent 把"该看什么、按什么顺序看、每步发生了什么"主动推到用户面前, 是 `from agent to agent` 在主体型产物上的落地。

## 规范本体

### 形态(技术栈 / 接口)

- **设施唯一**: 演示运行时 = wiki-core `demo` 模块, 不另起包。接口:
  - `mountDemoTour(rootEl, { tour, appRoot, comments, autoplay, onStep, reducedMotion })`(wiki-core `demo.js`, 类型权威 `demo-script.d.ts`)。
  - 评论存储 `createDemoCommentStore({ endpoint, materialId })`(wiki-core `comments.js` 兄弟函数), `target.kind='demo_step'`。
- **承载**: 审阅台把 app 的真实页面用 `html` 材料 + `extra.live_url` 同源 iframe 承载(复用现有 `HtmlMaterialView`), app 在 `?demo=<tourId>` 时自挂载覆盖层。**不新增 material kind、不新增后端端点。**
- **一份脚本两个出口**: 同一 `TourScript` 既驱动实时覆盖层, 又被 `tools/ops/ops-run-demo-tour.mjs`(克隆自 selfplay 录制器, 复用 `beat()`)跑出录制回退(`.webm`+帧+manifest), app 未起服时材料用 `file_relpath` 回退。**演示步骤只有一个真源(TourScript), 录制器不得另写步骤。**

### 格式(TourScript)

权威 schema = wiki-core `demo-script.d.ts`。每步字段:

| 字段 | 含义 |
|---|---|
| `id` | 稳定的步骤 id(评论锚点), 如 `r2x-01-setup` |
| `title` / `narration` | 覆盖层标题 / 叙述(中文) |
| `target` | CSS 选择器(优先复用已有 `data-testid`), 用于高亮 |
| `action` | 到达本步状态的声明式命令(click/clickCell/waitFor/waitMs/eval-hook), **纯 JSON, 不内联函数** |
| `screenshotRef` | 录制回退帧基名(与 ops 的 `beat` 命名对齐) |
| `links` | 出链(`doc_paragraph` / `material` / `demo_step`) |

### 目录 / 命名规范

- tour 注册表在 app 内: `src/.../demo-tours.ts`(随 app 走, 不散落到别处)。
- 录制产物落 `docs/reviews/<YYYY-MM-DD>-<slug>/`: `demo-<tourId>.webm`、`beat-*.png`、`demo-<tourId>-report.json`。
- 启动钩 `?demo=<tourId>` 与 `?dev=1`/`?devbattle=` 同族, 玩家面零泄漏(`clean=1` 时覆盖层仍可在上层)。

### 评论→审批→修改意见闭环

- 每步"评论"按钮 → `target={kind:'demo_step', tour_id, step_id, step_index, title}` → reviewstage `/{id}/comment`。
- "修改意见" = 读侧派生: `{评论 target.kind=='demo_step'} ∩ {material.status=='accepted'}`, 复用 `comment.feedback_status` 生命周期。**不新增存储字段。**

### 三大禁忌(散落 / 旁路 / 重复造轮)

- 不散落: 产物只落 `docs/reviews/<date>/`, registry 只在 app。
- 不旁路: 评论只走 reviewstage 评论 API, 锚点只用 FNV-1a(与 dashboard `annotations.ts` 一致)。
- 不重复造轮: 演示运行时并入 wiki-core, 录制复用 selfplay `beat()`, iframe 复用 `html`+`live_url`。

### 中文-only gate

覆盖层 `narration`/`title` 是用户可见文本, 受中文-only gate(禁英文与开发术语)。
